
<h1 align="center">平台需求分析文档Requirements Analysis</h1>



- Background/ Introduction

    - 随着大健康产业在全球范围内的快速发展，科研与产业的深度融合已成为推动技术突破和成果转化的关键动力。然而，在实际操作中，高校专家资源分散、信息不对称、合作沟通效率低等问题，依然是制约澳大利亚大健康领域协同创新的重要障碍。

    - 当前，企业希望寻找合适的科研合作伙伴，访问学者与硕博申请者希望高效对接导师，政府和科研机构也亟需建立可视化的专家图谱与合作网络。尽管已有部分高校官网或学术平台（如Google Scholar、LinkedIn）提供个体信息，但缺乏系统化、结构化、跨高校的统一平台。

    - 为此，我们提出开发一个面向大健康领域的全国性专家信息平台，整合全澳所有大学的大健康专家资源，搭建一站式信息检索、合作联络与智能推荐平台，助力科研与产业之间的桥梁构建，推动澳大利亚大健康领域的知识共享与技术转化。

- System Objectives

    本系统旨在通过数据整合与人工智能支持，实现以下核心目标：

    - 构建全面、结构化的大健康专家信息库，涵盖澳大利亚各大高校相关领域专家的研究方向、联系方式、项目经验及外部平台链接（如Google Scholar、GitHub、HuggingFace、LinkedIn等）；

    - 支持多维度、高精度的信息检索与筛选功能，基于一级和二级领域标签、机构、研究方向、关键词等进行搜索；

    - 提供一站式对接与合作意向发起功能，支持企业、研究者、访问学者、申请人等多方角色便捷发起合作请求；

    - 构建具备AI能力的智能匹配与推荐机制，通过引入语义搜索、嵌入向量匹配、专家知识图谱或多智能体协同等技术，支持根据研究方向、关键词、机构背景及合作需求实现专家资源的个性化推荐。平台具备良好的可扩展性，未来可结合大模型微调、LangGraph 等工作流引擎或 RAG 框架，进一步提升系统在垂直专业领域的理解力与推荐准确性；

    - 促进学术与产业之间的高效链接与成果转化，推动跨界协作与创新生态的形成。

- User roles & permissions

    本平台面向多类用户群体，不同用户在系统中承担不同的任务角色，访问不同的功能模块。系统需支持多角色并行、权限分级和可配置化的访问控制。

    - 平台管理员：负责平台整体运维与后台管理，包括用户权限配置、专家审核、数据更新、系统设置等。

    - 专家用户：平台数据库中的核心对象。专家可认领其个人页面，进行资料完善与更新，接收合作意向、参与互动反馈。

    - 企业/机构用户：来自产业界、政府机构、医院、科研单位等有合作需求的组织成员，可浏览专家资料、发起合作意向、提交咨询请求等。

    - 学术用户：包括硕博申请者、访问学者申请人、高校教职工等，主要用于查找潜在导师或合作团队，提交对接请求。

    - 访客用户：无需注册即可访问平台公开部分专家信息，但无法发起联系或访问特定内容。

    - 后续可拓展角色：

        - 内容编辑/数据运营： 协助平台管理员进行数据维护与内容发布

        - 专家助手/研究助理：可由专家邀请代为管理个人主页

        - API接入技术人员：对接其他平台（如高校系统、研究机构门户）所用的访问接口角色
    
    用户生命周期：系统需支持用户的注册、登录、身份认证（如实名或机构邮箱验证）、角色绑定、信息更新、账号注销等完整生命周期管理流程。

- Functional requirements

    - Expert directory module

        - 功能目标：提供一个结构化、全面、可检索的专家信息平台，支持专家页面展示、资料维护及后台审核。

        - 主要功能：

            - 展示全澳高校大健康专家信息，包括姓名、职称、所属机构、研究方向、联系方式、学术链接（学校个人主页、Google Scholar、ResearchGate、ORCID、LinkedIn、GitHub、HuggingFace等）

            - 支持按照一级/二级研究领域、机构、关键词等维度筛选

            - 支持专家认领并编辑个人页面（需身份认证）

            - 后台审核专家提交的修改请求（这个要考虑一下是否需要？）

            - 支持专家头像、代表性成果、合作项目等信息上传

    - Search& filtering module

        - 功能目标：实现对专家资源的高效、多维度检索，提升用户查找效率与精度。

        - 主要功能：

            - 关键词模糊匹配搜索

            - 多条件组合筛选：研究方向、地区、机构、合作意向、语言能力等

            - 支持标签体系：一级领域（如公共卫生、精准医疗）+ 二级子领域

            - 搜索结果分页展示、排序（如按活跃度、引用数、合作频率等）

    - AI match & recommendation module

        - 功能目标：利用人工智能辅助用户发现最合适的专家资源，实现个性化推荐与连接建议。

        - 主要功能：

            - 用户输入研究关键词或合作意图 → 系统推荐相关专家列表

            - 支持嵌入式语义匹配、知识图谱、专家合作关系分析

            - 多智能体框架实现对话式专家推荐体验

            - 支持模型微调/拓展以提升大健康领域匹配精度

            - 推荐结果提供可解释性标记（如匹配关键词/合作历史）

    - Collaboration request module

        - 功能目标：为企业、研究人员等用户提供直接对接专家的渠道，促进科研合作和资源对接。

        - 主要功能：

            - 用户可向专家发起“合作意向”请求，填写基本信息（单位、需求、预期合作方式等）

            - 专家接收请求提醒，可通过平台查看详情、选择接受/拒绝

            - 所有发起/响应行为记录于系统中，可供后台统计分析

            - 请求状态追踪与提醒功能（邮件或站内通知）

            - 可选设置隐私权限：允许公开邮箱/仅通过平台沟通

    - User & access management

        - 功能目标：支持多类型用户注册、身份验证、权限分配和生命周期管理。

        - 主要功能：

            - 用户注册与角色绑定（企业、专家、学术用户等）

            - 支持机构邮箱验证/专家身份认证机制

            - 支持第三方账号登录（如LinkedIn、Google）

            - 用户信息修改、找回密码、注销账户等基本功能

            - 管理员可配置角色权限、冻结/启用账户

    - Admin operations module

        - 功能目标：为平台管理员提供内容审核、数据维护、用户管理等后台功能。

        - 主要功能：

            - 专家信息库数据导入/导出/批量修改

            - 用户反馈与合作请求的统计与管理

            - 系统公告与站内通知发布

            - 数据报表与访问分析（如访问热度、搜索关键词趋势）

            - 权限管理与日志追踪

    - Analytics & visualization

        - 功能目标：帮助管理方了解平台使用情况，辅助科研管理与政策制定。

        - 主要功能：

            - 专家分布图（按高校、研究方向、地区）

            - 合作频率统计、领域热点分析

            - 用户行为路径分析、推荐命中率报告

    - Multilingual & internationalization support

        - 功能目标：支持平台面向不同语言背景用户（如中文/英文）使用，扩大国际访问者与华人科研人员的使用体验。

        - 主要功能：

            - 全站支持中英文切换，基于用户浏览器或个人设置自动切换语言

            - 支持专家资料中多语言描述字段（如项目简介、研究方向等）

            - 平台通知、按钮、搜索栏、邮件等关键交互元素多语言化

            - 支持未来拓展至更多语种（如印地语、阿拉伯语等）

    - Responsive & mobile support

        - 功能目标：确保平台在多终端下均能良好运行，适应不同用户使用场景。

        - 主要功能：

            - 响应式Web设计，兼容桌面、平板、手机等设备

            - 针对小屏幕设备优化搜索、筛选、阅读等交互体验

            - 可拓展为PWA（Progressive Web App），支持离线访问与本地缓存

            - 可评估未来是否开发原生 App（如 iOS/Android）

    - API integration module

        - 功能目标：提供统一的数据访问接口，支持高校/机构平台对接、专家数据同步及外部应用集成。

        - 主要功能：

            - 提供RESTful API接口，支持专家信息查询、推荐请求、合作发起等功能

            - 提供API文档与密钥认证机制

            - 支持与高校系统、政府科研平台、数据门户等系统集成

            - 可拓展为GraphQL、Webhook 等新式接入方式

    - Policy support & reporting module

        - 功能目标：为政府部门、高校科研处等提供定期可视化报告，支持科研决策、人才布局与成果转化政策支持。

        - 主要功能：

            - 支持按季度/年度生成平台使用统计与趋势分析报告

            - 季度报告内容可包含：专家分布、合作情况、访问频次、热门研究方向等

            - 年度报告更聚焦澳大利亚大健康领域国家竞争力、澳大利亚高等教育科研合作力、重点研究方向的产业需求匹配度评估、澳洲科研人才流动趋势与政策建议等等

            - 可扩展年度分析报告包括区域健康研究发展指数、海外访问学者流动趋势与促进建议、大健康研究热点分析、学术-产业联通效率评分模型等

            - 支持导出为PDF、PPT、Word等格式，适用于政府/合作单位审阅

            - 可配置报告模板与调度规则（如每年7月自动生成上一财年报告）


- Non-functional requirements

    - Performance requirements

        - 响应时间：在常规访问量下，页面加载时间应≤2秒，搜索和匹配结果响应时间≤3秒。

        - 并发处理能力：系统应支持至少同时500人在线访问，未来可扩展至1000人以上。

        - 数据处理能力：支持万级别专家信息的快速加载与筛选，AI匹配模块响应时间不超过5秒。

    - Availability & reliability

        - 系统可用性：平台应达到99.9%的年平均可用性，支持7x24小时运行。

        - 系统容错性：关键模块（如检索与匹配）应具备自动降级机制，避免全站故障。

        - 日志与监控：需建立错误日志、访问日志、行为日志系统，并对异常行为设置自动告警。

    - Security

        - 身份认证：采用OAuth2或多因素身份认证方式，确保账户安全。

        - 访问控制：基于用户角色的权限控制系统（RBAC），确保数据访问隔离。

        - 数据传输安全：平台必须使用HTTPS进行数据传输，防止中间人攻击。

        - 数据存储安全：敏感信息（如邮箱、身份验证信息）需加密存储，符合澳大利亚隐私法。

        - 防御机制：具备基础DDoS防护、SQL注入、XSS攻击检测能力。

    - Scalability

        - 系统应支持模块化架构设计，便于未来功能拓展（如加入API模块、多语言模块、AI扩展模块等）。

        - 支持横向扩展（通过负载均衡器部署多个服务实例）和纵向扩展（提高单节点计算/存储能力）。

    - 兼容性（Compatibility）

        - 支持主流浏览器（Chrome, Safari, Firefox, Edge）无障碍访问；

        - 页面应具备响应式设计，兼容桌面端、平板及移动端浏览；

        - 后台数据可导出为Excel/PDF格式，兼容政府及高校数据接收标准；

        - 与第三方平台（如Google Scholar、ORCID、LinkedIn、高校系统）具备良好接入兼容性。

    - Maintainability & Deployment

        - 采用标准化代码结构和文档规范，支持多人协作开发；

        - 后台管理模块应具备日志查看、内容审核、系统设置功能；

        - 提供定期备份机制与系统更新计划，支持自动化部署（CI/CD流程）；

        - 系统上线后需支持至少1年维护期，提供技术支持通道。

    - Legal & Compliance

        - 符合澳大利亚《隐私法》（Privacy Act 1988）关于个人信息采集与使用的规定；

        - 若数据涉及国际访问，需满足GDPR相关要求；

        - 支持用户数据导出、数据删除请求等权利行使接口；

        - 合作单位及专家信息需具备来源可追溯机制，避免违规爬取或展示。

- Development phases & timeline

    - MVP - expected 3 months

        - 目标：快速构建可运行平台原型，实现基本专家信息展示、搜索与AI智能匹配，支持基础用户注册与合作意向发起。

        - 交付功能模块：

            - 专家信息库（数据结构设计、页面展示）

            - 高级搜索与多维筛选（支持一级/二级领域、机构、关键词）

            - AI匹配引擎（基于关键词或研究方向的语义推荐）

            - 用户注册与角色体系（支持企业、学术用户、访客等）

            - 合作意向提交与专家提醒机制（基础表单通知）

            - 管理后台基础功能（内容审核、权限配置）

            - 响应式网页支持（桌面+移动浏览器）

    - V1.0正式上线版本 - 预计第4-6个月

        - 目标：优化用户体验，扩展AI能力，增强专家页面互动性与系统稳定性，实现平台对外展示与初步推广。

        - 新增功能：

            - 专家认领功能 + 编辑资料接口

            - 多智能体结构（如LangGraph）辅助问答/推荐逻辑

            - 平台内消息提醒、通知中心

            - 数据结构优化、搜索性能提升

            - 基础数据分析面板（访问量、专家热度等）

    - V2.0高级功能扩展 - 预计第7-10个月

        - 目标：拓展平台应用边界，实现移动适配、API对接、可视化分析及国际化支持，推动产学研合作机制落地。

        - 新增功能：

            - 多语言支持（中英切换）

            - 跨平台响应式体验优化（含PWA框架）

            - 外部API接口（供高校/科研系统调用）

            - 平台统计与合作可视化分析

            -  推荐结果的可解释性标签

    - V3.0 战略版 / 政策服务版 - 预计第10–12个月

        - 新增功能：

            - 年度/季度自动化统计报告生成 - 目标季度报告自动生成/年度报告作为科研项目撰写

            - 区域科研力、学科热度等图谱分析

            - 数据导出、报告模板配置功能

            - 政策建议报告撰写支持（自动化摘要、指标分析）

            - 高级图谱可视化（如合作网络图）

    - 后续优化与维护阶段（持续）

        - 持续优化用户体验与界面交互；

        - 引入商业模式模块（如项目对接服务、专家认证服务等）；

        - 技术架构升级与安全机制增强；

        - 推广与用户反馈闭环管理机制建立。

    - 开发方法建议（可选说明）

        - 采用敏捷开发（Agile）迭代推进，每2–3周一个sprint；

        - 使用DevOps流程进行版本部署与质量追踪；

        - 构建持续集成（CI）与自动化测试框架；

        - 建议采用模块化+微服务架构，便于不同功能并行开发与维护。
